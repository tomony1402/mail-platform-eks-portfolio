# TROUBLESHOOTING

## カスタムネットワーキング検討と断念

### やろうとしたこと

- Node 1台に複数 Pod をのせてそれぞれ別パブリック IP で配信
- コスト削減が目的

### AWSの壁

- ENI のセカンダリ IP にはパブリック IP を紐付けできない
- 結局 1 Node = 1 パブリック IP
- オンプレ（eth0 エイリアス）では `x.x.x.0/24` の IP を 1 台に複数持たせて配信できたが、AWS では不可能

### 結論

- Node 1台 = Pod 1台 = パブリック IP 1個 のシンプルな構成に落ち着いた
- Karpenter + Spot でノードを IP ローテーションしながら 60 台で配信
- オンプレの常識がクラウドで通用しなかった典型例
